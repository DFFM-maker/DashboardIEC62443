### FASE 2 — ASSET DISCOVERY E INVENTORY

Esegui i seguenti comandi su Kali via SSH e salva tutti gli output
in `~/assessment/raw/`.

**2a. Host discovery (ARP — non invasivo)**
```bash
sudo nmap -sn 172.16.224.0/20 \
  -oX ~/assessment/raw/discovery.xml \
  -oG ~/assessment/raw/discovery.txt
```

**2b. Port scan su porte OT critiche**
```bash
sudo nmap -sS \
  -p 21,22,23,25,80,102,443,502,515,4840,8080,9600,20000,44818,50000 \
  --max-rate 50 -T2 \
  --open \
  -iL <(grep "Up" ~/assessment/raw/discovery.txt | awk '{print $2}') \
  -oX ~/assessment/raw/portscan.xml
```

**2c. Service version detection**
```bash
sudo nmap -sV --version-intensity 7 \
  -p 80,443,4840,9600,44818,50000,8080 \
  --max-rate 30 -T2 \
  -iL <(grep "Up" ~/assessment/raw/discovery.txt | awk '{print $2}') \
  -oX ~/assessment/raw/versions.xml
```

**2d. Script NSE per protocolli OT**
```bash
# EtherNet/IP (Omron)
sudo nmap -sU -p 44818 --script enip-info \
  172.16.224.21,22,23,24 \
  -oX ~/assessment/raw/enip.xml

# FINS Omron
sudo nmap -p 9600 --script omron-info \
  172.16.224.21,22,23,24 \
  -oX ~/assessment/raw/fins.xml 2>/dev/null || \
  echo "Script omron-info non disponibile, skip"

# OPC-UA
sudo nmap -p 4840 --script opcua-info \
  172.16.224.21,22,23,24 \
  -oX ~/assessment/raw/opcua.xml 2>/dev/null || \
  echo "Script opcua-info non disponibile, skip"

# Modbus
sudo nmap -p 502 --script modbus-discover \
  172.16.224.0/20 \
  -oX ~/assessment/raw/modbus.xml
```

**2e. HTTP banner grab su Hakko HMI e device HTTP**
```bash
for ip in 172.16.224.41 172.16.224.42 172.16.224.43 172.16.224.44 172.16.239.2; do
  echo "=== $ip ===" >> ~/assessment/raw/http_banners.txt
  curl -s -v -m 5 http://$ip/ 2>&1 | head -50 >> ~/assessment/raw/http_banners.txt
  echo "" >> ~/assessment/raw/http_banners.txt
done
```

**2f. Banner grab B&R porta 50000**
```bash
for ip in 172.16.224.201 172.16.224.202 172.16.224.203 172.16.224.204; do
  echo "=== $ip:50000 ===" >> ~/assessment/raw/br_banners.txt
  echo "" | nc -v -w 3 $ip 50000 2>&1 | head -20 >> ~/assessment/raw/br_banners.txt
done
```

**2g. SNMP enumeration (se abilitato su Omron)**
```bash
for ip in 172.16.224.21 172.16.224.22 172.16.224.23 172.16.224.24; do
  echo "=== SNMP $ip ===" >> ~/assessment/raw/snmp.txt
  snmpwalk -v2c -c public -t 3 $ip 2>&1 | head -100 >> ~/assessment/raw/snmp.txt
  echo "" >> ~/assessment/raw/snmp.txt
done
```

**2h. Device non identificato .10 — probe esteso**
```bash
sudo nmap -Pn -sS -sU \
  -p T:1-1024,U:161,162,67,68,69,514 \
  --version-intensity 5 \
  172.16.224.10 \
  -oX ~/assessment/raw/unknown_10.xml
```

### FASE 3 — ANALISI DEI RISULTATI

Leggi tutti i file in `~/assessment/raw/` e costruisci le seguenti
strutture dati in memoria:

**3a. Asset Inventory Table**
Per ogni device trovato, estrai:
```
{
  "ip": "",
  "mac": "",
  "vendor": "",
  "device_type": "",      # PLC / HMI / Router / Unknown
  "device_model": "",     # da banner/service detection
  "firmware_version": "", # se disponibile
  "open_ports": [],       # lista {port, protocol, service, version}
  "protocols_ot": [],     # protocolli OT identificati
  "snmp_info": {},        # se disponibile
  "notes": ""
}
```

**3b. Vulnerability Findings**
Per ogni device, identifica i finding di sicurezza basandoti su:
- Porte aperte non necessarie (principio least privilege)
- Protocolli non cifrati (HTTP, FINS, EtherNet/IP non autenticato)
- Versioni software con CVE noti (cerca nei dati di version detection)
- Credenziali di default (testa solo se esplicitamente abilitato)
- Configurazioni insicure rilevabili passivamente

Per ogni finding:
```
{
  "finding_id": "F-001",
  "device_ip": "",
  "title": "",
  "description": "",
  "cvss_score": 0.0,        # stima CVSS v3.1
  "cvss_vector": "",
  "severity": "",           # Critical/High/Medium/Low/Info
  "iec62443_sr": [],        # lista SR violati es. ["SR 4.1", "SR 7.7"]
  "iec62443_part": "",      # es. "3-3"
  "evidence": "",           # output nmap/curl che supporta il finding
  "remediation": "",
  "remediation_priority": "" # Immediate/Short-term/Long-term
}
```

**3c. Zone e Conduit Map**
Analizza i pattern di comunicazione e le subnet per proporre:
- Divisione in Security Zones secondo IEC 62443-3-2
- Conduit necessari tra zone con protocolli identificati
- Security Level (SL) proposto per ogni zona (SL-1/2/3)

Logica suggerita per questo impianto:
```
Zone 1 — PLC Zone (SL-2)
  Assets: Omron .21-.24, B&R .201-.204
  Protocolli interni: EtherNet/IP, FINS, TCP/50000

Zone 2 — HMI Zone (SL-1)
  Assets: Hakko .41-.44
  Protocolli: HTTP

Zone 3 — Remote Access Zone (SL-2)
  Assets: Secomea .254
  Protocolli: HTTPS, 8080

Zone 4 — Unknown/Unclassified
  Assets: .10, HPE .239.2
  Da classificare dopo identificazione

Conduit C1: HMI Zone → PLC Zone
  Protocolli ammessi: FINS, EtherNet/IP
  Direzione: bidirezionale

Conduit C2: Remote Access Zone → PLC Zone
  Protocolli ammessi: HTTPS only
  Direzione: inbound con autenticazione
```

Adatta e correggi questa proposta in base a ciò che hai trovato
effettivamente durante la scansione.

## NOTE OPERATIVE

- Usa `sshpass -p kali ssh kali@172.16.224.250 "<comando>"` per tutti
  i comandi remoti — evita prompt interattivi durante l'automazione.
  Se sshpass non è installato su Windows, installalo prima:
  su Kali: `sudo apt install -y sshpass`
  oppure usa PowerShell con `plink.exe` di PuTTY:
  `plink -pw kali kali@172.16.224.250 "<comando>"`

- Se un comando fallisce, logga l'errore e continua con il successivo
- Se nmap trova device aggiuntivi non in lista, aggiungili all'inventory
- Non eseguire mai `--script vuln` senza conferma esplicita su device OT
- Non testare credenziali di default senza autorizzazione scritta
- Il Secomea SiteManager (.254) va scansionato solo con `-sV` gentile,
  mai con scan aggressivi che potrebbero interrompere la connettività remota
- Ogni comando eseguito va loggato con timestamp in `~/assessment/logs/commands.log`

## STANDARD DI RIFERIMENTO

- IEC 62443-3-2: Security risk assessment and system design
  → Guida la definizione di zone, conduit e security levels
- IEC 62443-3-3 SR 6.2: Continuous monitoring
  → Giustifica la necessità di inventario aggiornato
- IEC 62443-2-1: Security management system
  → Fornisce il framework per remediation e prioritizzazione
- CVSS v3.1: Common Vulnerability Scoring System
  → Base per scoring dei finding
